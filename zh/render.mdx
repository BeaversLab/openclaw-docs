title: 在 Render 上部署
---

使用基础架构即代码在 Render 上部署 OpenClaw。包含的 `render.yaml` Blueprint 以声明方式定义了您的整个堆栈，包括服务、磁盘、环境变量，以便您可以单击一键部署并将您的基础架构与代码一起进行版本控制。

## 先决条件

- 一个 [Render 账户](https://render.com)（提供免费层级）
- 来自您首选的[模型提供商](/en/providers)的 API 密钥

1. 根据此仓库根目录下的 `render.yaml` Blueprint 创建一个新的 Render 服务。
2. 提示您设置 `SETUP_PASSWORD`
3. 构建 Docker 镜像并部署

部署后，您的服务 URL 将遵循模式 `https://<service-name>.onrender.com`。

## 了解 Blueprint

Render Blueprints 是定义您的基础架构的 YAML 文件。此存储库中的 `render.yaml` 配置了运行 OpenClaw 所需的一切：

```yaml
services:
  - type: web
    name: openclaw
    runtime: docker
    plan: starter
    healthCheckPath: /health
    envVars:
      - key: PORT
        value: "8080"
      - key: SETUP_PASSWORD
        sync: false # prompts during deploy
      - key: OPENCLAW_STATE_DIR
        value: /data/.openclaw
      - key: OPENCLAW_WORKSPACE_DIR
        value: /data/workspace
      - key: OPENCLAW_GATEWAY_TOKEN
        generateValue: true # auto-generates a secure token
    disk:
      name: openclaw-data
      mountPath: /data
      sizeGB: 1
```

使用的关键 Blueprint 功能：

| 功能                  | 用途                                     |
| --------------------- | ---------------------------------------- |
| `runtime: docker`     | 从仓库的 Dockerfile 构建                 |
| `healthCheckPath`     | Render 监视 `/health` 并重启不健康的实例 |
| `sync: false`         | 在部署期间提示输入值（机密）             |
| `generateValue: true` | 自动生成一个加密安全的值                 |
| `disk`                | 在重新部署后仍然存在的持久存储           |

## 选择计划

| 计划         | 休眠           | 磁盘   | 最适合             |
| ------------ | -------------- | ------ | ------------------ |
| 免费         | 空闲 15 分钟后 | 不可用 | 测试、演示         |
| 入门版       | 从不           | 1GB+   | 个人使用、小团队   |
| 标准版及以上 | 从不           | 1GB+   | 生产环境、多个频道 |

该 Blueprint 默认为 `starter`。要使用免费层级，请在您的 fork 的 `render.yaml` 中更改 `plan: free`（但请注意：没有持久磁盘意味着每次部署时配置都会重置）。

## 部署后

### 完成设置向导

1. 导航到 `https://<your-service>.onrender.com/setup`
2. 输入您的 `SETUP_PASSWORD`
3. 选择一个模型提供商并粘贴您的 API 密钥
4. （可选）配置消息频道（Telegram、Discord、Slack）
5. 点击 **运行设置 (Run setup)**

### 访问控制 UI

Web 控制面板可通过 `https://<your-service>.onrender.com/openclaw` 访问。

## Render 控制面板功能

### 日志

在 **Dashboard → your service → Logs** 中查看实时日志。筛选方式：

- 构建日志（Docker 镜像创建）
- 部署日志（服务启动）
- 运行时日志（应用输出）

### Shell 访问

要进行调试，请通过 **Dashboard → your service → Shell** 打开一个 shell 会话。持久化磁盘挂载于 `/data`。

### 环境变量

在 **Dashboard → your service → Environment** 中修改变量。更改将触发自动重新部署。

### 自动部署

如果您使用原始的 OpenClaw 仓库，Render 将不会自动部署您的 OpenClaw。要更新它，请从控制面板手动运行 Blueprint 同步。

## 自定义域名

1. 前往 **Dashboard → your service → Settings → Custom Domains**
2. 添加您的域名
3. 按照说明配置 DNS（CNAME 指向 `*.onrender.com`）
4. Render 会自动配置 TLS 证书

## 扩缩容

Render 支持水平和垂直扩缩容：

- **垂直扩缩容**：更改计划以获取更多的 CPU/RAM
- **水平扩缩容**：增加实例数量（Standard 计划及以上）

对于 OpenClaw，垂直扩缩容通常已足够。水平扩缩容需要粘性会话或外部状态管理。

## 备份和迁移

随时导出您的配置和工作区：

```
https://<your-service>.onrender.com/setup/export
```

这将下载一个便携式备份，您可以在任何 OpenClaw 主机上恢复。

## 故障排除

### 服务无法启动

在 Render 控制面板中检查部署日志。常见问题：

- 缺少 `SETUP_PASSWORD` — Blueprint 会提示输入此项，但请检查其是否已设置
- 端口不匹配 — 确保 `PORT=8080` 与 Dockerfile 中暴露的端口匹配

### 冷启动缓慢（免费层）

免费层服务在闲置 15 分钟后会休眠。休眠后的首个请求需要几秒钟时间来启动容器。升级到 Starter 计划以保持常开。

### 重新部署后数据丢失

这发生在免费层（无持久化磁盘）。升级到付费计划，或
定期通过 `/setup/export` 导出您的配置。

### 健康检查失败

Render 期望在 30 秒内从 `/health` 收到 200 响应。如果构建成功但部署失败，服务启动时间可能过长。请检查：

- 检查构建日志中的错误
- 容器是否可以使用 `docker build && docker run` 在本地运行
