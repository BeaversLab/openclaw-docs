# Discord (机器人 API)

状态：可通过官方 Discord 机器人 Gateway 在私聊和服务器文本频道中使用。

## 快速设置（初学者）

1. 创建 Discord 机器人并复制机器人令牌。
2. 在 Discord 应用设置中，启用**消息内容意图**（如果你计划使用白名单或名称查找，还需要**服务器成员意图**）。
3. 为 OpenClaw 设置令牌：
   - 环境变量：%%P505%%
   - 或配置：%%P506%%。
   - 如果两者都设置了，配置优先（环境变量回退仅适用于默认账户）。
4. 使用消息权限将机器人邀请到你的服务器（如果只需要私聊，请创建一个私人服务器）。
5. 启动Gateway。
6. 私聊访问默认为配对模式；在首次联系时批准配对代码。

最小配置：

```json5
{
  channels: {
    discord: {
      enabled: true,
      token: "YOUR_BOT_TOKEN",
    },
  },
}
```

## 目标

- 通过 Discord 私聊或服务器频道与 OpenClaw 交谈。
- 私聊会话折叠到代理的主会话中（默认 %%P507%%）；服务器频道保持为 %%P508%%（显示名称使用 %%P509%%）。
- 群组私聊默认被忽略；通过 %%P510%% 启用，并可选择通过 %%P511%% 限制。
- 保持路由确定性：回复始终返回到它们到达的频道。

## 工作原理

1. 创建一个 Discord 应用 → 机器人，启用你需要的意图（私聊 + 服务器消息 + 消息内容），并获取机器人令牌。
2. 使用所需的权限将机器人邀请到你的服务器，以在你想要使用的地方读/发送消息。
3. 使用 %%P512%% 配置 OpenClaw（或使用 %%P513%% 作为回退）。
4. 运行Gateway；当令牌可用时（优先配置，回退环境变量）且 %%P514%% 不为 %%P515%% 时，它会自动启动 Discord 频道。
   - 如果你更喜欢环境变量，请设置 %%P516%%（配置块是可选的）。
5. 私聊：投递时使用 %%P517%%（或 %%P518%% 提及）；所有轮次都会落入共享的 %%P519%% 会话。纯数字 ID 有歧义，会被拒绝。
6. 服务器频道：使用 %%P520%% 进行投递。默认情况下需要提及，可以按服务器或按频道设置。
7. 私聊：通过 %%P521%% 默认安全（默认值：%%P522%%）。未知发送者会获得配对代码（1 小时后过期）；通过 %%P523%% 批准。
   - 要保持旧的”对任何人开放”行为：设置 %%P524%% 和 %%P525%%。
   - 要硬白名单：设置 %%P526%% 并在 %%P527%% 中列出发送者。
   - 要忽略所有私聊：设置 %%P528%% 或 %%P529%%。
8. 群组私聊默认被忽略；通过 %%P530%% 启用，并可选择通过 %%P531%% 限制。
9. 可选的服务器规则：设置 %%P532%%，以服务器 ID（首选）或 slug 为键，包含每个频道的规则。
10. 可选的原生命令：%%P533%% 默认为 %%P534%%（Discord/Telegram 为 on，Slack 为 off）。使用 %%P535%% 覆盖；%%P536%% 清除先前注册的命令。文本命令由 %%P537%% 控制，必须作为独立的 %%P538%% 消息发送。使用 %%P539%% 绕过命令的访问组检查。
    - 完整命令列表 + 配置：[Slash commands]%%P540%%
11. 可选的服务器上下文历史记录：设置 %%P541%%（默认 20，回退到 %%P542%%）以在回复提及时包含最后 N 条服务器消息作为上下文。设置 %%P543%% 以禁用。
12. 反应：代理可以通过 %%P544%% 工具触发反应（由 %%P545%% 控制）。
    - 反应移除语义：请参阅 [/tools/reactions]%%P547%%。
    - %%P546%% 工具仅在当前频道为 Discord 时暴露。
13. 原生命令使用隔离的会话键（%%P548%%），而不是共享的 %%P549%% 会话。

注意：名称 → id 解析使用服务器成员搜索，需要服务器成员意图；如果机器人无法搜索成员，请使用 id 或 %%P550%% 提及。
注意：Slug 是小写的，空格替换为 %%P551%%。频道名称经过 slug 处理，不带前导 %%P552%%。
注意：服务器上下文 %%P553%% 行包括 %%P554%% + %%P555%%，以便轻松生成 ping 准备的回复。

## 配置写入

默认情况下，允许 Discord 写入由 %%P556%% 触发的配置更新（需要 %%P557%%）。

禁用方法：

```json5
{
  channels: { discord: { configWrites: false } },
}
```

## 如何创建你自己的机器人

这是”Discord 开发者门户”设置，用于在服务器（公会）频道（如 %%P558%%）中运行 OpenClaw。

### 1) 创建 Discord 应用 + 机器人用户

1. Discord 开发者门户 → **应用程序** → **新建应用程序**
2. 在你的应用程序中：
   - **机器人** → **添加机器人**
   - 复制**机器人令牌**（这是你放入 %%P559%% 的内容）

### 2) 启用 OpenClaw 需要的Gateway意图

Discord 会阻止”特权意图”，除非你明确启用它们。

在 **机器人** → **特权Gateway意图**中，启用：

- **消息内容意图**（在大多数服务器中读取消息文本所必需；没有它你会看到”使用了不允许的意图”或机器人会连接但不对消息做出反应）
- **服务器成员意图**（推荐；某些成员/用户查找和服务器中的白名单匹配所必需）

你通常**不**需要 **Presence 意图**。设置机器人自己的在线状态（%%P560%% 操作）使用Gateway OP3，不需要此意图；仅当你想要接收其他服务器成员的在线状态更新时才需要。

### 3) 生成邀请 URL（OAuth2 URL 生成器）

在你的应用程序中：**OAuth2** → **URL 生成器**

**作用域**

- ✅ %%P561%%
- ✅ %%P562%%（原生命令所必需）

**机器人权限**（最低基准）

- ✅ 查看频道
- ✅ 发送消息
- ✅ 读取消息历史
- ✅ 嵌入链接
- ✅ 附加文件
- ✅ 添加反应（可选但推荐）
- ✅ 使用外部表情符号/贴纸（可选；仅在需要时）

避免使用 **Administrator**，除非你在调试并且完全信任机器人。

复制生成的 URL，打开它，选择你的服务器，然后安装机器人。

### 4) 获取 ID（服务器/用户/频道）

Discord 在任何地方都使用数字 ID；OpenClaw 配置首选 ID。

1. Discord（桌面/网页版）→ **用户设置** → **高级** → 启用 **开发者模式**
2. 右键单击：
   - 服务器名称 → **复制服务器 ID**（服务器 ID）
   - 频道（例如 %%P563%%）→ **复制频道 ID**
   - 你的用户 → **复制用户 ID**

### 5) 配置 OpenClaw

#### 令牌

通过环境变量设置机器人令牌（在服务器上推荐）：

- `DISCORD_BOT_TOKEN=...`

或通过配置：

```json5
{
  channels: {
    discord: {
      enabled: true,
      token: "YOUR_BOT_TOKEN",
    },
  },
}
```

多账户支持：使用 %%P564%% 配置每个账户的令牌和可选的 %%P565%%。有关共享模式，请参阅 [%%P566%%]%%P567%%。

#### 白名单 + 频道路由

示例”单服务器，仅允许我，仅允许 #help”：

```json5
{
  channels: {
    discord: {
      enabled: true,
      dm: { enabled: false },
      guilds: {
        YOUR_GUILD_ID: {
          users: ["YOUR_USER_ID"],
          requireMention: true,
          channels: {
            help: { allow: true, requireMention: true },
          },
        },
      },
      retry: {
        attempts: 3,
        minDelayMs: 500,
        maxDelayMs: 30000,
        jitter: 0.1,
      },
    },
  },
}
```

注意事项：

- %%P568%% 表示机器人仅在被提及时回复（推荐用于共享频道）。
- %%P569%%（或 %%P570%%）也算作服务器消息的提及。
- 多代理覆盖：在 %%P571%% 上设置每个代理的模式。
- 如果存在 %%P572%%，默认情况下未列出的任何频道都会被拒绝。
- 使用 %%P573%% 频道条目在所有频道上应用默认值；显式频道条目会覆盖通配符。
- 线程继承父频道配置（白名单、%%P574%%、技能、提示等），除非你明确添加线程频道 ID。
- 机器人编写的消息默认被忽略；设置 %%P575%% 以允许它们（自己的消息仍然被过滤）。
- 警告：如果你允许对其他机器人的回复（%%P576%%），请使用 %%P577%%、%%P578%% 白名单和/或在 %%P579%% 和 %%P580%% 中设置清晰的防护措施来防止机器人到机器人的回复循环。

### 6) 验证其工作正常

1. 启动Gateway。
2. 在你的服务器频道中，发送：%%P581%%（或你的机器人名称）。
3. 如果没有任何反应：检查下面的**故障排除**。

### 故障排除

- 首先：运行 %%P582%% 和 %%P583%%（可操作的警告 + 快速审计）。
- **”使用了不允许的意图”**：在开发者门户中启用 **消息内容意图**（可能还需要 **服务器成员意图**），然后重启Gateway。
- **机器人连接但在服务器频道中从不回复**：
  - 缺少 **消息内容意图**，或
  - 机器人缺少频道权限（查看/发送/读取历史），或
  - 你的配置需要提及而你没有提及它，或
  - 你的服务器/频道白名单拒绝了频道/用户。
- **%%P584%% 但仍然没有回复**：
- %%P585%% 默认为 **allowlist**；将其设置为 %%P586%% 或在 %%P587%% 下添加服务器条目（可选择在 %%P588%% 下列出频道以进行限制）。
  - 如果你只设置 %%P589%% 而从不创建 %%P590%% 部分，运行时会将 %%P591%% 默认为 %%P592%%。添加 %%P593%%、%%P594%% 或服务器/频道白名单以锁定它。
- %%P595%% 必须位于 %%P596%%（或特定频道）之下。顶级的 %%P597%% 被忽略。
- **权限审计**（%%P598%%）仅检查数字频道 ID。如果你使用 slug/名称作为 %%P599%% 键，审计无法验证权限。
- **私聊不工作**：%%P600%%、%%P601%%，或者你尚未被批准（%%P602%%）。
- **Discord 中的执行批准**：Discord 支持私聊中执行批准的**按钮界面**（允许一次 / 始终允许 / 拒绝）。%%P603%% 仅用于转发的批准，不会解决 Discord 的按钮提示。如果你看到 %%P604%% 或 UI 从未出现，请检查：
  - 你的配置中的 %%P605%%。
  - 你的 Discord 用户 ID 列在 %%P606%% 中（UI 仅发送给批准者）。
  - 使用私聊提示中的按钮（**允许一次**、**始终允许**、**拒绝**）。
  - 有关更广泛的批准和命令流程，请参阅 [执行批准]%%P607%% 和 [斜杠命令]%%P608%%。

## 功能和限制

- 私聊和服务器文本频道（线程被视为单独的频道；不支持语音）。
- 尽最大努力发送输入指示器；消息分块使用 %%P609%%（默认 2000），并按行数（%%P610%%，默认 17）分割长回复。
- 可选的换行符分块：设置 %%P611%% 以在长度分块之前在空行（段落边界）上分割。
- 支持文件上传，最大为配置的 %%P612%%（默认 8 MB）。
- 默认情况下，服务器回复需要提及门控以避免嘈杂的机器人。
- 当消息引用另一条消息时，会注入回复上下文（引用内容 + ID）。
- 原生回复线程**默认关闭**；使用 %%P613%% 和回复标签启用。

## 重试策略

出站 Discord API 调用在使用 Discord %%P614%% 时在速率限制（429）上重试（如果可用），具有指数退避和抖动。通过 %%P615%% 配置。请参阅 [重试策略]%%P616%%。

### PluralKit (PK) 支持

启用 PK 查找，以便代理消息解析到底层系统 + 成员。
启用后，OpenClaw 将成员身份用于白名单，并将发送者标记为 %%P716%% 以避免意外的 Discord ping。

```json5
{
  channels: {
    discord: {
      pluralkit: {
        enabled: true,
        token: "pk_live_...", // optional; required for private systems
      },
    },
  },
}
```

白名单注意事项（启用 PK）：

- 在 %%P718%%、%%P719%% 或每个频道的 %%P720%% 中使用 %%P717%%。
- 成员显示名称也按名称/slug 匹配。
- 查找使用**原始** Discord 消息 ID（代理前的消息），因此 PK API 仅在其 30 分钟窗口内解析它。
- 如果 PK 查找失败（例如，没有令牌的私有系统），代理消息将被视为机器人消息并被丢弃，除非 %%P721%%。

## 回复标签

要请求线程回复，模型可以在其输出中包含一个标签：

- %%P726%% — 回复触发的 Discord 消息。
- %%P727%% — 回复来自上下文/历史的特定消息 ID。

## 安全和运维

- 将机器人令牌视为密码；在受监控的主机上优先使用 %%P784%% 环境变量或锁定配置文件权限。
- 仅授予机器人所需的权限（通常是读取/发送消息）。
- 如果机器人卡住或受到速率限制，在确认没有其他进程拥有 Discord 会话后，重启Gateway（%%P785%%）。
