# 模型配置（探索）

本文档记录了未来模型配置的**构想**。它并非
已发布的规范。关于当前行为，请参阅：

- [模型](/zh/concepts/models)
- [模型故障转移](/zh/concepts/model-failover)
- [OAuth + 配置文件](/zh/concepts/oauth)

## 动机

操作员希望：

- 每个提供商拥有多个身份验证配置文件（个人与工作）。
- 简单的 `/model` 选择，具有可预测的回退机制。
- 在文本模型和具备图像处理能力的模型之间明确区分。

## 可能的方向（高层级）

- 保持模型选择简单：使用带有可选别名的 `provider/model`。
- 允许提供商拥有多个身份验证配置文件，并具有明确的顺序。
- 使用全局回退列表，以便所有会话都能一致地进行故障转移。
- 仅在明确配置时才覆盖图像路由。

## 未决问题

- 配置文件的轮换应该是基于提供商还是基于模型？
- UI 应该如何为会话展示配置文件选择？
- 从旧版配置键迁移的最安全路径是什么？
