---
title: 在 Railway 上部署
---

使用一键模板在 Railway 上部署 OpenClaw，并在浏览器中完成设置。
这是最简单的“服务器上无终端”路径：Railway 为您运行网关，
您可以通过 `/setup` 网络向导配置所有内容。

## 快速检查清单（新用户）

1. 点击 **Deploy on Railway**（下方）。
2. 添加挂载于 `/data` 的 **Volume**。
3. 设置所需的 **Variables**（至少 `SETUP_PASSWORD`）。
4. 在端口 `8080` 上启用 **HTTP Proxy**。
5. 打开 `https://<your-railway-domain>/setup` 并完成向导。

## 一键部署

<a href="https://railway.com/deploy/clawdbot-railway-template" target="_blank" rel="noreferrer">
  在 Railway 上部署
</a>

部署后，在 **Railway → 您的服务 → Settings → Domains** 中找到您的公共 URL。

Railway 将会：

- 为您提供一个生成的域名（通常是 `https://<something>.up.railway.app`），或者
- 如果您绑定了自定义域名，则使用该域名。

然后打开：

- `https://<your-railway-domain>/setup` — 设置向导（受密码保护）
- `https://<your-railway-domain>/openclaw` — 控制界面

## 您将获得

- 托管的 OpenClaw 网关 + 控制界面
- 位于 `/setup` 的 Web 设置向导（无需终端命令）
- 通过 Railway 卷 (`/data`) 提供的持久存储，确保配置/凭据/工作空间在重新部署后得以保留
- 位于 `/setup/export` 的备份导出功能，以便日后从 Railway 迁移

## 必需的 Railway 设置

### 公共网络

为该服务启用 **HTTP Proxy**。

- 端口：`8080`

### 卷（必需）

附加一个挂载于以下路径的卷：

- `/data`

### 变量

在服务上设置这些变量：

- `SETUP_PASSWORD`（必需）
- `PORT=8080`（必需 —— 必须与公网网络设置中的端口相匹配）
- `OPENCLAW_STATE_DIR=/data/.openclaw`（推荐）
- `OPENCLAW_WORKSPACE_DIR=/data/workspace`（推荐）
- `OPENCLAW_GATEWAY_TOKEN`（推荐；请将其作为管理员密钥处理）

## 设置流程

1. 访问 `https://<your-railway-domain>/setup` 并输入您的 `SETUP_PASSWORD`。
2. 选择一个模型/身份验证提供商并粘贴您的密钥。
3. （可选）添加 Telegram/Discord/Slack 令牌。
4. 点击 **运行设置 (Run setup)**。

如果 Telegram 私信设置为配对模式，设置向导可以批准配对码。

## 获取聊天令牌

### Telegram 机器人令牌

1. 在 Telegram 中给 `@BotFather` 发消息
2. 运行 `/newbot`
3. 复制 token（看起来像 `123456789:AA...`）
4. 将其粘贴到 `/setup` 中

### Discord bot token

1. 前往 https://discord.com/developers/applications
2. **New Application** → 选择一个名称
3. **Bot** → **Add Bot**
4. 在 Bot → Privileged Gateway Intents 下启用 **MESSAGE CONTENT INTENT**（必须启用，否则 bot 会在启动时崩溃）
5. 复制 **Bot Token** 并粘贴到 `/setup` 中
6. 将 bot 邀请到您的服务器（OAuth2 URL Generator；scopes: `bot`, `applications.commands`）

## 备份与迁移

在以下地址下载备份：

- `https://<your-railway-domain>/setup/export`

这将导出您的 OpenClaw 状态 + 工作区，以便您迁移到其他主机而不丢失配置或记忆。
