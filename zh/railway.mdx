---
title: "在 Railway 上部署"
---

通过一键模板在 Railway 上部署 OpenClaw，并在浏览器中完成设置。
这是最简单的”无需在服务器上使用终端”的路径：Railway 为您运行 Gateway，
您可以通过 %%P1%% 网页向导配置所有内容。

## 快速检查清单（新用户）

1. 点击 **Deploy on Railway**（下方）。
2. 添加挂载在 %%P2%% 的 **Volume**。
3. 设置所需的 **Variables**（至少 %%P3%%）。
4. 在端口 %%P4%% 上启用 **HTTP Proxy**。
5. 打开 %%P5%% 并完成向导。

## 一键部署

<a href="https://railway.com/deploy/clawdbot-railway-template" target="_blank" rel="noreferrer">
  Deploy on Railway
</a>

部署后，在 **Railway → 您的服务 → Settings → Domains** 中找到您的公共 URL。

Railway 将会：

- 为您提供一个生成的域名（通常是 %%P6%%），或
- 如果您附加了自定义域名，则使用您的自定义域名。

然后打开：

- %%P7%% — 设置向导（受密码保护）
- %%P8%% — 控制 UI

## 您将获得

- 托管的 OpenClaw Gateway + 控制 UI
- 位于 %%P9%% 的网页设置向导（无需终端命令）
- 通过 Railway Volume (%%P10%%) 实现持久化存储，使配置/凭据/工作空间在重新部署后得以保留
- 位于 %%P11%% 的备份导出，以便日后从 Railway 迁移

## 设置流程

1. 访问 %%P18%% 并输入您的 %%P19%%。
2. 选择模型/身份验证提供商并粘贴您的密钥。
3. （可选）添加 Telegram/Discord/Slack 令牌。
4. 点击 **Run setup**。

如果 Telegram 私信设置为配对模式，设置向导可以批准配对代码。

## 获取聊天令牌

### Telegram 机器人令牌

1. 在 Telegram 中向 %%P20%% 发送消息
2. 运行 %%P21%%
3. 复制令牌（看起来像 %%P22%%）
4. 将其粘贴到 %%P23%% 中

### Discord 机器人令牌

1. 访问 https://discord.com/developers/applications
2. **New Application** → 选择一个名称
3. **Bot** → **Add Bot**
4. 在 Bot → Privileged Gateway Intents 下启用 **MESSAGE CONTENT INTENT**（必需，否则机器人将在启动时崩溃）
5. 复制 **Bot Token** 并粘贴到 %%P24%% 中
6. 邀请机器人到您的服务器（OAuth2 URL Generator；范围：%%P25%%、%%P26%%）

## 备份与迁移

在以下位置下载备份：

- `https://<your-railway-domain>/setup/export`

这将导出您的 OpenClaw 状态 + 工作空间，以便您可以迁移到另一个主机而不会丢失配置或记忆。
