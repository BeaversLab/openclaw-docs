# 为 OpenClaw 威胁模型做贡献

感谢您帮助让 OpenClaw 更安全。该威胁模型是一份持续更新的文档，我们欢迎任何人的贡献——您无需成为安全专家。

## 贡献方式

### 添加威胁

发现了我们未涵盖的攻击向量或风险？请在 [openclaw/trust](https://github.com/openclaw/trust/issues) 上提出 issue，并用您自己的话对其进行描述。您无需了解任何框架或填写每个字段——只需描述场景即可。

**最好包含（但非必需）：**

- 攻击场景及其可能被利用的方式
- 受影响的 OpenClaw 部分（CLI、网关、频道、ClawHub、MCP 服务器等）
- 您认为的严重程度（低 / 中 / 高 / 严重）
- 任何相关研究、CVE 或真实示例的链接

我们将在审查期间处理 ATLAS 映射、威胁 ID 和风险评估。如果您想包含这些细节，那很好——但这并不是预期的。

> **这是为了添加到威胁模型中，而不是报告实时漏洞。** 如果您发现了可利用的漏洞，请参阅我们的 [信任页面](https://trust.openclaw.ai) 了解负责任的披露说明。

### 建议缓解措施

关于如何解决现有威胁有什么想法？请提出 issue 或 PR 以引用该威胁。有用的缓解措施是具体且可操作的——例如，“在网关处实施每发送者 10 条消息/分钟的速率限制”要比“实施速率限制”更好。

### 提出攻击链

攻击链展示了多种威胁如何组合成现实的攻击场景。如果您发现了某种危险的组合，请描述步骤以及攻击者如何将它们串联起来。一段关于攻击在实践中如何展开的简短叙述比正式的模板更有价值。

### 修复或改进现有内容

拼写错误、澄清内容、过时信息、更好的示例——欢迎提交 PR，无需提出 issue。

## 我们使用的工具

### MITRE ATLAS

此威胁模型基于 [MITRE ATLAS](https://atlas.mitre.org/)（对抗性威胁景观针对 AI 系统），这是一个专门针对 AI/ML 威胁（如提示注入、工具滥用和代理利用）设计的框架。您无需了解 ATLAS 即可参与贡献——我们会在审查期间将提交内容映射到该框架。

### 威胁 ID

每个威胁都有一个类似 `T-EXEC-003` 的 ID。类别包括：

| 代码    | 类别                |
| ------- | ------------------- |
| RECON   | 侦察 - 信息收集     |
| ACCESS  | 初始访问 - 获得入口 |
| EXEC    | 执行 - 运行恶意操作 |
| PERSIST | 持久化 - 维持访问   |
| EVADE   | 防御规避 - 避免检测 |
| DISC    | 设备发现 - 了解环境 |
| EXFIL   | 数据渗出 - 窃取数据 |
| IMPACT  | 影响 - 损害或中断   |

ID 由维护人员在审查期间分配。您无需选择一个。

### 风险级别

| 级别     | 含义                                      |
| -------- | ----------------------------------------- |
| **严重** | 系统完全受损，或高可能性 + 严重影响       |
| **高**   | 可能造成重大损害，或中等可能性 + 严重影响 |
| **中等** | 中等风险，或低可能性 + 高影响             |
| **低**   | 可能性低且影响有限                        |

如果您不确定风险级别，只需描述影响，我们将对其进行评估。

## 审查流程

1. **分类** - 我们会在 48 小时内审查新的提交内容
2. **评估** - 我们验证可行性，分配 ATLAS 映射和威胁 ID，确认风险级别
3. **文档** - 我们确保所有内容格式正确且完整
4. **合并** - 添加到威胁模型和可视化中
