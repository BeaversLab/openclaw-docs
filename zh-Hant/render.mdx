---
title: 在 Render 上部署
---

使用基礎設施即程式碼在 Render 上部署 OpenClaw。包含的 `render.yaml` 藍圖以宣告方式定義了您的整個技術堆疊、服務、磁碟和環境變數，因此您可以透過單擊進行部署，並將您的基礎設施與程式碼一起進行版本控制。

## 先決條件

- 一個 [Render 帳戶](https://render.com)（提供免費層級）
- 來自您首選 [模型提供者](/zh-Hant/providers) 的 API 金鑰

## 使用 Render 藍圖進行部署

<a
  href="https://render.com/deploy?repo=https://github.com/openclaw/openclaw"
  target="_blank"
  rel="noreferrer"
>
  部署到 Render
</a>

點擊此連結將：

1. 根據此儲存庫根目錄下的 `render.yaml` 藍圖建立一個新的 Render 服務。
2. 提示您設定 `SETUP_PASSWORD`
3. 建置 Docker 映像檔並進行部署

部署完成後，您的服務 URL 將遵循 `https://<service-name>.onrender.com` 模式。

## 了解藍圖

Render 藍圖是定義基礎設施的 YAML 檔案。本儲存庫中的 `render.yaml`
設定檔配置了執行 OpenClaw 所需的一切：

```yaml
services:
  - type: web
    name: openclaw
    runtime: docker
    plan: starter
    healthCheckPath: /health
    envVars:
      - key: PORT
        value: "8080"
      - key: SETUP_PASSWORD
        sync: false # prompts during deploy
      - key: OPENCLAW_STATE_DIR
        value: /data/.openclaw
      - key: OPENCLAW_WORKSPACE_DIR
        value: /data/workspace
      - key: OPENCLAW_GATEWAY_TOKEN
        generateValue: true # auto-generates a secure token
    disk:
      name: openclaw-data
      mountPath: /data
      sizeGB: 1
```

使用的關鍵藍圖功能：

| 功能                  | 用途                                             |
| --------------------- | ------------------------------------------------ |
| `runtime: docker`     | 從儲存庫的 Dockerfile 建置                       |
| `healthCheckPath`     | Render 監控 `/health` 並重新啟動不正常的執行個體 |
| `sync: false`         | 在部署期間提示輸入值（秘密）                     |
| `generateValue: true` | 自動產生加密安全的值                             |
| `disk`                | 在重新部署後仍然存在的持久儲存空間               |

## 選擇方案

| 方案       | 休眠           | 磁碟     | 最適用於           |
| ---------- | -------------- | -------- | ------------------ |
| 免費       | 閒置 15 分鐘後 | 無法使用 | 測試、展示         |
| 入門       | 永不           | 1GB+     | 個人使用、小型團隊 |
| 標準及以上 | 永不           | 1GB+     | 生產環境、多個頻道 |

藍圖預設為 `starter`。若要使用免費層級，請修改您分叉版本中的 `plan: free`
在 `render.yaml` 中（但請注意：沒有永久磁碟意味著每次部署時設定都會重置）。

## 部署完成後

### 完成設定精靈

1. 導航至 `https://<your-service>.onrender.com/setup`
2. 輸入您的 `SETUP_PASSWORD`
3. 選擇模型提供者並貼上您的 API 金鑰
4. 選擇性配置訊息通道（Telegram、Discord、Slack）
5. 點擊 **執行設定**

### 存取控制介面

網頁儀表板位於 `https://<your-service>.onrender.com/openclaw`。

## Render 儀表板功能

### 日誌

在 **儀表板 → 您的服務 → 日誌** 中檢視即時日誌。篩選條件：

- 建置日誌（Docker 映像檔建立）
- 部署日誌（服務啟動）
- 執行時期日誌（應用程式輸出）

### Shell 存取

若要進行除錯，請透過 **儀表板 → 您的服務 → Shell** 開啟 Shell 工作階段。持久磁碟已掛載於 `/data`。

### 環境變數

在 **儀表板 → 您的服務 → 環境** 中修改變數。變更會觸發自動重新部署。

### 自動部署

如果您使用原始的 OpenClaw 儲存庫，Render 將不會自動部署您的 OpenClaw。若要更新它，請從儀表板執行手動 Blueprint 同步。

## 自訂網域

1. 前往 **儀表板 → 您的服務 → 設定 → 自訂網域**
2. 新增您的網域
3. 依照指示設定 DNS（CNAME 至 `*.onrender.com`）
4. Render 會自動佈建 TLS 憑證

## 擴展

Render 支援水平和垂直擴展：

- **垂直**：變更方案以獲得更多 CPU/RAM
- **水平**：增加執行個體數量（標準方案及以上）

對於 OpenClaw，垂直擴展通常足夠。水平擴展需要連線會議或者外部狀態管理。

## 備份與移轉

隨時匯出您的設定和工作區：

```
https://<your-service>.onrender.com/setup/export
```

這會下載一個可攜式備份，您可以在任何 OpenClaw 主機上還原。

## 疑難排解

### 服務無法啟動

檢查 Render 儀表板中的部署日誌。常見問題：

- 遺失 `SETUP_PASSWORD` — Blueprint 會提示輸入此項，但請驗證是否已設定
- 連接埠不符 — 確保 `PORT=8080` 符合 Dockerfile 中公開的連接埠

### 冷啟動緩慢（免費方案）

免費方案的服務在閒置 15 分鐘後會進入休眠。休眠後的首次請求在容器啟動時需要幾秒鐘。升級至 Starter 方案即可保持常時運作。

### 重新部署後資料遺失

這會發生在免費層級（無永久磁碟）。請升級至付費方案，或
透過 `/setup/export` 定期匯出您的設定。

### 健康檢查失敗

Render 預期在 30 秒內從 `/health` 收到 200 回應。如果建置成功但部署失敗，服務可能啟動時間過長。請檢查：

- 建置記錄中的錯誤
- 容器是否能在本機使用 `docker build && docker run` 執行
