使用基礎設施即程式碼在 Render 上部署 OpenClaw。包含的 `render.yaml` Blueprint 以宣告式方式定義了您的整個堆疊、服務、磁碟、環境變數，因此您可以透過單擊進行部署，並將您的基礎設施與程式碼一起進行版本控制。

## 先決條件

- 一個 [Render 帳戶](https://render.com) （提供免費層級）
- 來自您首選 [模型提供者](/zh-Hant/providers) 的 API 金鑰

## 使用 Render Blueprint 進行部署

[部署到 Render](https://render.com/deploy?repo=https://github.com/openclaw/openclaw)

點擊此連結將：

1. 根據此儲存庫根目錄中的 `render.yaml` Blueprint 建立新的 Render 服務。
2. 提示您設定 `SETUP_PASSWORD`
3. 建置 Docker 映像檔並部署

部署完成後，您的服務 URL 將遵循 `https://<service-name>.onrender.com` 的模式。

## 了解藍圖

Render 藍圖是用來定義基礎架構的 YAML 檔案。本儲存庫中的 `render.yaml` 設定了執行 OpenClaw 所需的一切：

```yaml
services:
  - type: web
    name: openclaw
    runtime: docker
    plan: starter
    healthCheckPath: /health
    envVars:
      - key: PORT
        value: "8080"
      - key: SETUP_PASSWORD
        sync: false # prompts during deploy
      - key: OPENCLAW_STATE_DIR
        value: /data/.openclaw
      - key: OPENCLAW_WORKSPACE_DIR
        value: /data/workspace
      - key: OPENCLAW_GATEWAY_TOKEN
        generateValue: true # auto-generates a secure token
    disk:
      name: openclaw-data
      mountPath: /data
      sizeGB: 1
```

使用的主要藍圖功能：

| 功能                  | 用途                                             |
| --------------------- | ------------------------------------------------ |
| `runtime: docker`     | 從儲存庫的 Dockerfile 建置                       |
| `healthCheckPath`     | Render 監控 `/health` 並重新啟動不健康的執行個體 |
| `sync: false`         | 部署時提示輸入值 (機密)                          |
| `generateValue: true` | 自動產生密碼學安全的值                           |
| `disk`                | 在重新部署後持續存在的儲存空間                   |

## 選擇方案

| 方案  | 休眠           | 磁碟     | 最適合             |
| ----- | -------------- | -------- | ------------------ |
| 免費  | 閒置 15 分鐘後 | 無法使用 | 測試、演示         |
| 入門  | 永不           | 1GB+     | 個人使用、小型團隊 |
| 標準+ | 永不           | 1GB+     | 生產環境、多個頻道 |

藍圖 預設為 `starter`。若要使用免費層級，請在您 fork 的
`render.yaml` 中變更 `plan: free`（但請注意：沒有永久磁碟代表每次部署時設定會重置）。

## 部署完成後

### 完成網頁設定

1. 導覽至 `https://<your-service>.onrender.com/setup`
2. 輸入您的 `SETUP_PASSWORD`
3. 選擇模型供應商並貼上您的 API 金鑰
4. 選擇性設定訊息頻道（Telegram、Discord、Slack）
5. 點擊 **執行設定**

### 存取控制介面

網頁儀表板位於 `https://<your-service>.onrender.com/openclaw`。

## Render 儀表板功能

### 日誌

在 **儀表板 → 您的服務 → 日誌** 中檢視即時日誌。過濾條件：

- 建置日誌（Docker 映像檔建立）
- 部署日誌（服務啟動）
- Runtime logs (application output)

### Shell access

For debugging, open a shell session via **Dashboard → your service → Shell**. The persistent disk is mounted at `/data`.

### Environment variables

Modify variables in **Dashboard → your service → Environment**. Changes trigger an automatic redeploy.

### Auto-deploy

If you use the original OpenClaw repository, Render will not auto-deploy your OpenClaw. To update it, run a manual Blueprint sync from the dashboard.

## Custom domain

1. Go to **Dashboard → your service → Settings → Custom Domains**
2. Add your domain
3. Configure DNS as instructed (CNAME to `*.onrender.com`)
4. Render provisions a TLS certificate automatically

## Scaling

Render supports horizontal and vertical scaling:

- **Vertical**: Change the plan to get more CPU/RAM
- **Horizontal**: Increase instance count (Standard plan and above)

對於 OpenClaw，垂直擴展通常就足夠了。水平擴展需要粘性會話或外部狀態管理。

## 備份與遷移

您可以隨時匯出您的配置和工作區：

```
https://<your-service>.onrender.com/setup/export
```

這會下載一個可移植的備份，您可以在任何 OpenClaw 主機上還原它。

## 疑難排解

### 服務無法啟動

請檢查 Render Dashboard 中的部署日誌。常見問題包括：

- 缺少 `SETUP_PASSWORD` — Blueprint 會提示輸入此項，但請確認已設定
- 連接埠不匹配 — 請確保 `PORT=8080` 與 Dockerfile 中公開的連接埠相符

### 冷啟動緩慢（免費方案）

免費方案的服務在閒置 15 分鐘後會進入休眠。休眠後的第一次請求需要幾秒鐘時間來啟動容器。請升級至 Starter 方案以保持常時運作。

### 重新部署後資料遺失

這發生在免費層級（無持久磁碟）。升級到付費方案，或
定期透過 `/setup/export` 匯出您的設定。

### 健康檢查失敗

Render 預期在 30 秒內從 `/health` 收到 200 回應。如果建置成功但部署失敗，服務可能啟動過久。請檢查：

- 建置日誌中的錯誤
- 容器是否可以在本機透過 `docker build && docker run` 執行
