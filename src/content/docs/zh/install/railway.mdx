使用一键模板在 Railway 上部署 OpenClaw，并通过 Web 控制 UI 进行访问。
这是最简单的“服务器上无需终端”路径：Railway 为您运行 Gateway(网关)。

## 快速清单（新用户）

1. 点击 **Deploy on Railway**（下方）。
2. 添加一个挂载于 `/data` 的 **Volume（卷）**。
3. 设置所需的 **变量**（至少包括 `OPENCLAW_GATEWAY_PORT` 和 `OPENCLAW_GATEWAY_TOKEN`）。
4. 在端口 `8080` 上启用 **HTTP 代理**。
5. 打开 `https://<your-railway-domain>/openclaw` 并使用配置的共享密钥进行连接。此模板默认使用 `OPENCLAW_GATEWAY_TOKEN`；如果您将其替换为密码认证，请改用该密码。

## 一键部署

<a href="https://railway.com/deploy/clawdbot-railway-template" target="_blank" rel="noreferrer">
  在 Railway 上部署
</a>

部署后，在 **Railway → 您的服务 → Settings → Domains** 中找到您的公共 URL。

Railway 将会：

- 为您提供一个生成的域名（通常是 `https://<something>.up.railway.app`），或者
- 如果您绑定了自定义域名，则使用该域名。

然后打开：

- `https://<your-railway-domain>/openclaw` — 控制界面

## 您将获得

- 托管的 OpenClaw Gateway(网关) + 控制界面
- 通过 Railway Volume (`/data`) 进行持久化存储，以便 `openclaw.json`、
  每个代理的 `auth-profiles.json`、渠道/提供商状态、会话以及
  工作区在重新部署后得以保留

## 所需的 Railway 设置

### 公共网络

为该服务启用 **HTTP 代理**。

- 端口： `8080`

### 卷（必需）

挂载一个位于以下路径的卷：

- `/data`

### 变量

在服务上设置这些变量：

- `OPENCLAW_GATEWAY_PORT=8080` (必需 — 必须与 Public Networking 中的端口匹配)
- `OPENCLAW_GATEWAY_TOKEN` (必需；请将其视为管理员机密)
- `OPENCLAW_STATE_DIR=/data/.openclaw` (推荐)
- `OPENCLAW_WORKSPACE_DIR=/data/workspace` (推荐)

## 连接渠道

使用位于 `/openclaw` 的控制 UI，或通过 Railway 的 shell 运行 `openclaw onboard` 以获取渠道设置说明：

- [Telegram](/en/channels/telegram) (最快 — 只需一个 Bot Token)
- [Discord](/en/channels/discord)
- [所有渠道](/en/channels)

## 备份与迁移

导出您的状态、配置、认证档案和工作区：

```bash
openclaw backup create
```

这将创建一个包含 OpenClaw 状态以及任何已配置
工作区的可移植备份存档。详情请参阅 [备份](/en/cli/backup)。
