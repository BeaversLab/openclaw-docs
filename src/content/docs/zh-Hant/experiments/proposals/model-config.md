# 模型配置（探索）

本文件記錄了未來模型配置的**構想**。這並非發佈規格。若要了解目前的行為，請參閱：

- [模型](/zh-Hant/concepts/models)
- [模型容錯移轉](/zh-Hant/concepts/model-failover)
- [OAuth + 設定檔](/zh-Hant/concepts/oauth)

## 動機

操作員希望：

- 每個供應商有多個驗證設定檔（個人與工作）。
- 簡單的 `/model` 選擇，並具有可預測的後備機制。
- 文字模型與支援影像的模型之間有明確的區隔。

## 可能的方向（高階）

- 保持模型選擇簡單：`provider/model` 搭配選用別名。
- 讓供應商擁有多個驗證設定檔，並具有明確的順序。
- 使用全域後備清單，讓所有工作階段都一致地進行容錯移轉。
- 僅在明確設定時覆寫影像路由。

## 開放性問題

- 設定檔輪替應以供應商為單位還是以模型為單位？
- UI 應如何顯示工作階段的設定檔選擇？
- 從舊版設定金鑰遷移的最安全途徑為何？
