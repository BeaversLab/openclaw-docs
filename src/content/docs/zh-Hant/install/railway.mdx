# Railway

使用一鍵範本在 Railway 上部署 OpenClaw，並透過網頁控制 UI 存取。
這是最簡單的「伺服器上無需終端機」路徑：Railway 會為您執行閘道。

## 快速檢查清單（新使用者）

1. 點擊 **Deploy on Railway**（下方）。
2. 新增掛載於 `/data` 的 **Volume**。
3. 設定所需的 **Variables**（至少包含 `OPENCLAW_GATEWAY_PORT` 和 `OPENCLAW_GATEWAY_TOKEN`）。
4. 在連接埠 `8080` 啟用 **HTTP Proxy**。
5. 開啟 `https://<your-railway-domain>/openclaw` 並使用設定的共用金鑰進行連線。此範本預設使用 `OPENCLAW_GATEWAY_TOKEN`；如果您將其替換為密碼驗證，請改用該密碼。

## 一鍵部署

<a href="https://railway.com/deploy/clawdbot-railway-template" target="_blank" rel="noreferrer">
  在 Railway 上部署
</a>

部署完成後，在 **Railway → 您的服務 → Settings → Domains** 中尋找您的公開 URL。

Railway 將會：

- 提供一個產生的網域（通常是 `https://<something>.up.railway.app`），或是
- 如果您附加了自訂網域，則使用該網域。

然後開啟：

- `https://<your-railway-domain>/openclaw` — Control UI

## 您將獲得

- 託管的 OpenClaw Gateway + Control UI
- 透過 Railway Volume (`/data`) 提供的持久儲存空間，使 `openclaw.json`、
  每個代理程式的 `auth-profiles.json`、頻道/提供者狀態、工作階段和
  工作區在重新部署後得以保留

## 必要的 Railway 設定

### 公開網路

為該服務啟用 **HTTP Proxy**。

- 連接埠：`8080`

### Volume（必要）

附加一個掛載於以下位置的 Volume：

- `/data`

### 變數

在服務上設定這些變數：

- `OPENCLAW_GATEWAY_PORT=8080`（必要 — 必須與公開網路中的連接埠相符）
- `OPENCLAW_GATEWAY_TOKEN`（必要；請將其視為管理員密鑰）
- `OPENCLAW_STATE_DIR=/data/.openclaw`（建議）
- `OPENCLAW_WORKSPACE_DIR=/data/workspace`（建議）

## 連接頻道

使用位於 `/openclaw` 的 Control UI，或透過 Railway 的 shell 執行 `openclaw onboard` 以取得頻道設定說明：

- [Telegram](/zh-Hant/channels/telegram)（最快 — 只需要機器人權杖）
- [Discord](/zh-Hant/channels/discord)
- [所有頻道](/zh-Hant/channels)

## 備份與遷移

匯出您的狀態、設定、認證設定檔以及工作區：

```bash
openclaw backup create
```

這會建立一個可攜式備份封存檔，其中包含 OpenClaw 狀態以及所有已設定
的工作區。詳情請參閱 [Backup](/zh-Hant/cli/backup)。
