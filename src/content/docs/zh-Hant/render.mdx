---
title: 在 Render 上部署
---

使用基礎架構即代碼在 Render 上部署 OpenClaw。包含的 `render.yaml` 藍圖以聲明方式定義了您的整個堆疊，包括服務、磁碟、環境變量，因此您可以單擊進行部署，並將您的基礎架構與代碼一起進行版本控制。

## 先決條件

- 一個 [Render 帳戶](https://render.com)（提供免費層級）
- 來自您首選 [模型提供者](/zh-Hant/providers) 的 API 金鑰

## 使用 Render 藍圖進行部署

<a href="https://render.com/deploy?repo=https://github.com/openclaw/openclaw" target="_blank" rel="noreferrer">
  部署到 Render
</a>

點擊此連結將：

1. 根據此儲存庫根目錄下的 `render.yaml` 藍圖建立一個新的 Render 服務。
2. 提示您設定 `SETUP_PASSWORD`
3. 建置 Docker 映像檔並部署

部署完成後，您的服務 URL 將遵循 `https://<service-name>.onrender.com` 的模式。

## 了解藍圖

Render 藍圖是用於定義基礎架構的 YAML 檔案。此儲存庫中的 `render.yaml` 配置了運行 OpenClaw 所需的一切：

```yaml
services:
  - type: web
    name: openclaw
    runtime: docker
    plan: starter
    healthCheckPath: /health
    envVars:
      - key: PORT
        value: "8080"
      - key: SETUP_PASSWORD
        sync: false # prompts during deploy
      - key: OPENCLAW_STATE_DIR
        value: /data/.openclaw
      - key: OPENCLAW_WORKSPACE_DIR
        value: /data/workspace
      - key: OPENCLAW_GATEWAY_TOKEN
        generateValue: true # auto-generates a secure token
    disk:
      name: openclaw-data
      mountPath: /data
      sizeGB: 1
```

使用的關鍵藍圖功能：

| 功能                  | 用途                                         |
| --------------------- | -------------------------------------------- |
| `runtime: docker`     | 從儲存庫的 Dockerfile 建置                   |
| `healthCheckPath`     | Render 監控 `/health` 並重新啟動不正常的實例 |
| `sync: false`         | 在部署期間提示輸入值（機密）                 |
| `generateValue: true` | 自動生成密碼學安全的值                       |
| `disk`                | 在重新部署後仍然保留的持久儲存               |

## 選擇方案

| 方案       | 休眠           | 磁碟   | 適用於             |
| ---------- | -------------- | ------ | ------------------ |
| 免費       | 閒置 15 分鐘後 | 不可用 | 測試、演示         |
| 入門       | 從不           | 1GB+   | 個人使用、小團隊   |
| 標準及以上 | 從不           | 1GB+   | 生產環境、多個頻道 |

藍圖預設為 `starter`。若要使用免費層級，請在您 fork 的 `render.yaml` 中更改 `plan: free`（但請注意：沒有持久磁碟意味著每次部署都會重置配置）。

## 部署後

### 完成設定精靈

1. 導航至 `https://<your-service>.onrender.com/setup`
2. 輸入您的 `SETUP_PASSWORD`
3. 選擇一個模型提供者並貼上您的 API 金鑰
4. 選擇性地配置訊息頻道（Telegram、Discord、Slack）
5. 點擊 **執行設定**

### 存取控制介面

網頁儀表板可在 `https://<your-service>.onrender.com/openclaw` 取得。

## Render 儀表板功能

### 日誌

在 **儀表板 → 您的服務 → 日誌** 中檢視即時日誌。過濾依據：

- 建置日誌（Docker 映像檔建立）
- 部署日誌（服務啟動）
- 執行階段日誌（應用程式輸出）

### Shell 存取

為了進行除錯，請透過 **儀表板 → 您的服務 → Shell** 開啟 shell 工作階段。持久磁碟會掛載於 `/data`。

### 環境變數

在 **儀表板 → 您的服務 → 環境** 中修改變數。變更會觸發自動重新部署。

### 自動部署

如果您使用原始的 OpenClaw 存儲庫，Render 將不會自動部署您的 OpenClaw。若要更新它，請從儀表板執行手動 Blueprint 同步。

## 自訂網域

1. 前往 **儀表板 → 您的服務 → 設定 → 自訂網域**
2. 新增您的網域
3. 按照指示設定 DNS（CNAME 指向 `*.onrender.com`）
4. Render 會自動提供 TLS 憑證

## 擴展

Render 支援水平和垂直擴展：

- **垂直**：變更方案以獲得更多 CPU/RAM
- **水平**：增加執行個體數量（標準方案及以上）

對於 OpenClaw，垂直擴展通常就足夠了。水平擴展需要 sticky sessions 或外部狀態管理。

## 備份與遷移

隨時匯出您的設定和工作區：

```
https://<your-service>.onrender.com/setup/export
```

這會下載一個可攜式備份，您可以在任何 OpenClaw 主機上還原。

## 疑難排解

### 服務無法啟動

檢查 Render 儀表板中的部署日誌。常見問題：

- 缺少 `SETUP_PASSWORD` — Blueprint 會提示輸入此項，但請驗證是否已設定
- 連接埠不符 — 確認 `PORT=8080` 符合 Dockerfile 中公開的連接埠

### 冷啟動緩慢（免費方案）

免費方案的服務在閒置 15 分鐘後會進入休眠。休眠後的首次請求在容器啟動時會花費幾秒鐘。升級至 Starter 方案以保持常時運作。

### 重新部署後資料遺失

這發生在免費方案上（無持久磁碟）。升級至付費方案，或
定期透過 `/setup/export` 匯出您的設定。

### 健康檢查失敗

Render 預期在 30 秒內從 `/health` 收到 200 回應。如果建置成功但部署失敗，服務可能是啟動時間過長。請檢查：

- 檢查建置紀錄是否有錯誤
- 容器是否能在本機透過 `docker build && docker run` 執行
