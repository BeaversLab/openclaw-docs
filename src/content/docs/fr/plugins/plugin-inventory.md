# Inventaire des plugins

Cette page est générée à partir de `extensions/*/package.json`, `openclaw.plugin.json`npm,
et des exclusions du package npm racine `files`. Régénérez-la avec :

```bash
pnpm plugins:inventory:gen
```

## Définitions

- **Package npm cœur :** intégré dans le package npm npm`openclaw`npm et disponible sans installation de plugin séparée.
- **Package externe officiel :** plugin maintenu par OpenClaw omis du package npm cœur, conservé dans cet inventaire officiel, et installé à la demande via ClawHub et/ou npm.
- **Extraction de source uniquement :** plugin local au référentiel omis des artefacts npm publiés et non annoncé comme package installable.

Les extractions de source sont différentes des installations npm : après npm`pnpm install`, les plugins
bundlés se chargent depuis `extensions/<id>`, ce qui permet les modifications locales et les dépendances d'espace de travail locales au package.

## Installer un plugin

Utilisez la colonne **Distribution** pour décider si une installation est nécessaire. Les plugins qui
indiquent `included in OpenClaw`Gateway sont déjà présents dans le package cœur. Les packages
externes officiels nécessitent une installation, puis un redémarrage de la Gateway.

Par exemple, Discord est un package externe officiel :

```bash
openclaw plugins install @openclaw/discord
openclaw gateway restart
openclaw plugins inspect discord --runtime --json
```

Les spécifications de package nues essaient d'abord ClawHub, puis npm en repli. Pour forcer une source, utilisez
ClawHubnpm`clawhub:@openclaw/discord` ou `npm:@openclaw/discord`Discord. Après l'installation, suivez
la documentation de configuration du plugin, telle que [Discord](/fr/channels/discord), pour ajouter les identifiants
et la configuration du canal. Voir [Gérer les plugins](/fr/plugins/manage-plugins) pour les commandes de mise à jour,
de désinstallation et de publication.
