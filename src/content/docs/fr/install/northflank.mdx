# Northflank

Déployez OpenClaw sur Northflank avec un modèle en un clic et accédez-y via l'interface Web de Contrôle.
C'est le chemin le plus simple « sans terminal sur le serveur » : Northflank exécute la Gateway pour vous.

## Comment démarrer

1. Cliquez sur [Déployer OpenClaw](https://northflank.com/stacks/deploy-openclaw) pour ouvrir le modèle.
2. Créez un [compte sur Northflank](https://app.northflank.com/signup) si vous n'en avez pas déjà un.
3. Cliquez sur **Déployer OpenClaw maintenant**.
4. Définissez la variable d'environnement requise : `OPENCLAW_GATEWAY_TOKEN` (utilisez une valeur aléatoire forte).
5. Cliquez sur **Déployer la stack** pour construire et exécuter le modèle OpenClaw.
6. Attendez que le déploiement soit terminé, puis cliquez sur **Voir les ressources**.
7. Ouvrez le service OpenClaw.
8. Ouvrez l'URL publique OpenClaw sur `/openclaw` et connectez-vous en utilisant le secret partagé configuré. Ce modèle utilise `OPENCLAW_GATEWAY_TOKEN` par défaut ; si vous le remplacez par une authentification par mot de passe, utilisez plutôt ce mot de passe.

## Ce que vous obtenez

- OpenClaw hébergé + Gateway + Interface de Contrôle
- Stockage persistant via Volume Northflank (`/data`) pour que `openclaw.json`,
  `auth-profiles.json` par agent, l'état des channel/provider, les sessions et
  l'espace de travail survivent aux redéploiements

## Connecter un channel

Utilisez l'interface de Contrôle sur `/openclaw` ou exécutez `openclaw onboard` via SSH pour obtenir les instructions de configuration du channel :

- [Telegram](/fr/channels/telegram) (le plus rapide — juste un jeton de bot)
- [Discord](/fr/channels/discord)
- [Tous les channels](/fr/channels)
