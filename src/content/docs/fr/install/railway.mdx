Déployez OpenClaw sur Railway avec un modèle en un clic et accédez-y via l'interface web de contrôle.
C'est le chemin le plus simple « sans terminal sur le serveur » : Railway exécute le Gateway pour vous.

## Liste de contrôle rapide (nouveaux utilisateurs)

1. Cliquez sur **Déployer sur Railway** (ci-dessous).
2. Ajoutez un **Volume** monté sur `/data`.
3. Définissez les **Variables** requises (au moins `OPENCLAW_GATEWAY_PORT` et `OPENCLAW_GATEWAY_TOKEN`).
4. Activez le **Proxy HTTP** sur le port `8080`.
5. Ouvrez `https://<your-railway-domain>/openclaw` et connectez-vous en utilisant votre `OPENCLAW_GATEWAY_TOKEN`.

## Déploiement en un clic

<a href="https://railway.com/deploy/clawdbot-railway-template" target="_blank" rel="noreferrer">
  Déployer sur Railway
</a>

Après le déploiement, trouvez votre URL publique dans **Railway → votre service → Paramètres → Domaines**.

Railway effectuera soit :

- vous fournir un domaine généré (souvent `https://<something>.up.railway.app`), ou
- utiliser votre domaine personnalisé si vous en avez attaché un.

Ensuite, ouvrez :

- `https://<your-railway-domain>/openclaw` — Interface de contrôle

## Ce que vous obtenez

- Gateway OpenClaw hébergé + Interface de contrôle
- Stockage persistant via un Volume Railway (`/data`) afin que la configuration/les identifiants/l'espace de travail survivent aux redéploiements

## Paramètres Railway requis

### Mise en réseau publique

Activez le **proxy HTTP** pour le service.

- Port : `8080`

### Volume (requis)

Attachez un volume monté à :

- `/data`

### Variables

Définissez ces variables sur le service :

- `OPENCLAW_GATEWAY_PORT=8080` (requis — doit correspondre au port dans la mise en réseau publique)
- `OPENCLAW_GATEWAY_TOKEN` (requis ; à traiter comme un secret administrateur)
- `OPENCLAW_STATE_DIR=/data/.openclaw` (recommandé)
- `OPENCLAW_WORKSPACE_DIR=/data/workspace` (recommandé)

## Connecter un channel

Utilisez l'interface de contrôle sur `/openclaw` ou exécutez `openclaw onboard` via le shell de Railway pour les instructions de configuration du channel :

- [Telegram](/fr/channels/telegram) (le plus rapide — juste un jeton de bot)
- [Discord](/fr/channels/discord)
- [Tous les canaux](/fr/channels)

## Sauvegardes et migration

Exportez votre configuration et votre espace de travail :

```bash
openclaw backup create
```

Cela crée une archive de sauvegarde portable que vous pouvez restaurer sur n'importe quel hôte OpenClaw. Voir [Sauvegarde](/fr/cli/backup) pour plus de détails.
