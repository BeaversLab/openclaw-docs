# Railway

Déployez OpenClaw sur Railway avec un modèle en un clic et accédez-y via l'interface de contrôle web.
C'est le chemin le plus simple « sans terminal sur le serveur » : Railway exécute le Gateway pour vous.

## Liste de contrôle rapide (nouveaux utilisateurs)

1. Cliquez sur **Deploy on Railway** (ci-dessous).
2. Ajoutez un **Volume** monté sur `/data`.
3. Définissez les **Variables** requises (au moins `OPENCLAW_GATEWAY_PORT` et `OPENCLAW_GATEWAY_TOKEN`).
4. Activez le **Proxy HTTP** sur le port `8080`.
5. Ouvrez `https://<your-railway-domain>/openclaw` et connectez-vous en utilisant le secret partagé configuré. Ce modèle utilise `OPENCLAW_GATEWAY_TOKEN` par défaut ; si vous le remplacez par une authentification par mot de passe, utilisez plutôt ce mot de passe.

## Déploiement en un clic

<a href="https://railway.com/deploy/clawdbot-railway-template" target="_blank" rel="noreferrer">
  Deploy on Railway
</a>

Après le déploiement, trouvez votre URL publique dans **Railway → your service → Settings → Domains**.

Railway soit :

- vous donnera un domaine généré (souvent `https://<something>.up.railway.app`), ou
- utilisera votre domaine personnalisé si vous en avez attaché un.

Ensuite, ouvrez :

- `https://<your-railway-domain>/openclaw` — Control UI

## Ce que vous obtenez

- OpenClaw Gateway hébergé + Control UI
- Stockage persistant via Railway Volume (`/data`) pour que `openclaw.json`,
  `auth-profiles.json` par agent, l'état du canal/fournisseur, les sessions et
  l'espace de travail survivent aux redéploiements

## Paramètres Railway requis

### Réseau public

Activez le **Proxy HTTP** pour le service.

- Port : `8080`

### Volume (requis)

Attachez un volume monté sur :

- `/data`

### Variables

Définissez ces variables sur le service :

- `OPENCLAW_GATEWAY_PORT=8080` (requis — doit correspondre au port dans Réseau public)
- `OPENCLAW_GATEWAY_TOKEN` (requis ; traitez-le comme un secret administrateur)
- `OPENCLAW_STATE_DIR=/data/.openclaw` (recommandé)
- `OPENCLAW_WORKSPACE_DIR=/data/workspace` (recommandé)

## Connecter un channel

Utilisez l'interface de contrôle sur `/openclaw` ou exécutez `openclaw onboard` via le shell de Railway pour les instructions de configuration du channel :

- [Telegram](/fr/channels/telegram) (le plus rapide — juste un jeton de bot)
- [Discord](/fr/channels/discord)
- [Tous les channels](/fr/channels)

## Sauvegardes et migration

Exportez votre état, votre configuration, vos profils d'authentification et votre espace de travail :

```bash
openclaw backup create
```

Cela crée une archive de sauvegarde portable avec l'état OpenClaw ainsi que tout espace de travail configuré. Voir [Sauvegarde](/fr/cli/backup) pour plus de détails.
