# Inventario de complementos

Esta página se genera desde `extensions/*/package.json`, `openclaw.plugin.json`,
y las exclusiones del paquete npm raíz `files`. Regenerarla con:

```bash
pnpm plugins:inventory:gen
```

## Definiciones

- **Paquete npm central:** integrado en el paquete npm `openclaw` y disponible sin una instalación de complemento por separado.
- **Paquete externo oficial:** complemento mantenido por OpenClaw que se omite del paquete npm central, se mantiene en este inventario oficial y se instala a pedido través de ClawHub y/o npm.
- **Solo checkout de código fuente:** complemento local del repositorio que se omite de los artefactos npm publicados y no se anuncia como un paquete instalable.

Los checkouts de código fuente son diferentes de las instalaciones de npm: después de `pnpm install`, los complementos
distribuidos se cargan desde `extensions/<id>` por lo que las ediciones locales y las dependencias del espacio de trabajo local del paquete
están disponibles.

## Instalar un complemento

Use la columna **Distribución** para decidir si se necesita la instalación. Los complementos que
dicen `included in OpenClaw` ya están presentes en el paquete central. Los paquetes
externos oficiales necesitan una instalación y luego un reinicio del Gateway.

Por ejemplo, Discord es un paquete externo oficial:

```bash
openclaw plugins install @openclaw/discord
openclaw gateway restart
openclaw plugins inspect discord --runtime --json
```

Las especificaciones de paquete básicas intentan ClawHub primero, luego la alternativa de npm. Para forzar un origen, use
`clawhub:@openclaw/discord` o `npm:@openclaw/discord`. Después de la instalación, siga
la documentación de configuración del complemento, como [Discord](/es/channels/discord), para agregar credenciales
y configuración de canal. Consulte [Administrar complementos](/es/plugins/manage-plugins) para comandos de actualización,
desinstalación y publicación.
