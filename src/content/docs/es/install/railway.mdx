# Railway

Implementa OpenClaw en Railway con una plantilla de un clic y accede a través de la interfaz de usuario web de Control.
Esta es la ruta más fácil "sin terminal en el servidor": Railway ejecuta el Gateway por ti.

## Lista de verificación rápida (nuevos usuarios)

1. Haz clic en **Deploy on Railway** (abajo).
2. Añade un **Volume** montado en `/data`.
3. Establece las **Variables** requeridas (al menos `OPENCLAW_GATEWAY_PORT` y `OPENCLAW_GATEWAY_TOKEN`).
4. Habilita **HTTP Proxy** en el puerto `8080`.
5. Abre `https://<your-railway-domain>/openclaw` y conéctate usando el secreto compartido configurado. Esta plantilla usa `OPENCLAW_GATEWAY_TOKEN` por defecto; si lo reemplazas con autenticación por contraseña, usa esa contraseña en su lugar.

## Implementación con un clic

<a href="https://railway.com/deploy/clawdbot-railway-template" target="_blank" rel="noreferrer">
  Desplegar en Railway
</a>

Después de la implementación, encuentra tu URL pública en **Railway → tu servicio → Settings → Domains**.

Railway ya sea:

- te dará un dominio generado (a menudo `https://<something>.up.railway.app`), o
- usará tu dominio personalizado si adjuntaste uno.

Luego abre:

- `https://<your-railway-domain>/openclaw` — Interfaz de Control

## Lo que obtienes

- OpenClaw Gateway alojado + Interfaz de Control
- Almacenamiento persistente a través de Railway Volume (`/data`) para que `openclaw.json`,
  `auth-profiles.json` por agente, estado de canal/proveedor, sesiones y
  espacio de trabajo sobrevivan a las reimplementaciones

## Configuración requerida de Railway

### Redes públicas

Habilita **HTTP Proxy** para el servicio.

- Puerto: `8080`

### Volumen (requerido)

Adjunta un volumen montado en:

- `/data`

### Variables

Establece estas variables en el servicio:

- `OPENCLAW_GATEWAY_PORT=8080` (requerido — debe coincidir con el puerto en Redes públicas)
- `OPENCLAW_GATEWAY_TOKEN` (requerido; trátalo como un secreto de administrador)
- `OPENCLAW_STATE_DIR=/data/.openclaw` (recomendado)
- `OPENCLAW_WORKSPACE_DIR=/data/workspace` (recomendado)

## Conectar un canal

Usa la Interfaz de Control en `/openclaw` o ejecuta `openclaw onboard` a través del shell de Railway para obtener instrucciones de configuración del canal:

- [Telegram](/es/channels/telegram) (el más rápido — solo un token de bot)
- [Discord](/es/channels/discord)
- [Todos los canales](/es/channels)

## Copias de seguridad y migración

Exporte su estado, configuración, perfiles de autenticación y espacio de trabajo:

```bash
openclaw backup create
```

Esto crea un archivo de copia de seguridad portátil con el estado de OpenClaw más cualquier espacio de trabajo configurado. Consulte [Backup](/es/cli/backup) para obtener más detalles.
