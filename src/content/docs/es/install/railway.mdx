Implemente OpenClaw en Railway con una plantilla de un clic y acceda a él a través de la interfaz de usuario web de Control.
Esta es la ruta más fácil "sin terminal en el servidor": Railway ejecuta el Gateway por usted.

## Lista de verificación rápida (nuevos usuarios)

1. Haga clic en **Deploy on Railway** (Implementar en Railway) (abajo).
2. Añada un **Volume** (Volumen) montado en `/data`.
3. Establezca las **Variables** requeridas (al menos `OPENCLAW_GATEWAY_PORT` y `OPENCLAW_GATEWAY_TOKEN`).
4. Habilite el **Proxy HTTP** en el puerto `8080`.
5. Abra `https://<your-railway-domain>/openclaw` y conéctese usando su `OPENCLAW_GATEWAY_TOKEN`.

## Implementación de un solo clic

<a href="https://railway.com/deploy/clawdbot-railway-template" target="_blank" rel="noreferrer">
  Desplegar en Railway
</a>

Después de la implementación, encuentre su URL pública en **Railway → su servicio → Settings → Domains**.

Railway hará lo siguiente:

- le proporcionan un dominio generado (a menudo `https://<something>.up.railway.app`), o
- usará su dominio personalizado si adjuntó uno.

Luego abra:

- `https://<your-railway-domain>/openclaw` — Interfaz de usuario de Control

## Lo que obtienes

- Gateway de OpenClaw alojado + Interfaz de usuario de Control
- Almacenamiento persistente a través de Railway Volume (`/data`) para que la configuración/credenciales/espacio de trabajo sobrevivan a los re-despliegues

## Configuración requerida de Railway

### Redes públicas

Habilite el **Proxy HTTP** para el servicio.

- Puerto: `8080`

### Volumen (requerido)

Adjunte un volumen montado en:

- `/data`

### Variables

Establezca estas variables en el servicio:

- `OPENCLAW_GATEWAY_PORT=8080` (requerido — debe coincidir con el puerto en Redes públicas)
- `OPENCLAW_GATEWAY_TOKEN` (requerido; trátelo como un secreto de administrador)
- `OPENCLAW_STATE_DIR=/data/.openclaw` (recomendado)
- `OPENCLAW_WORKSPACE_DIR=/data/workspace` (recomendado)

## Conectar un canal

Use la interfaz de usuario de Control en `/openclaw` o ejecute `openclaw onboard` a través del shell de Railway para obtener instrucciones de configuración del canal:

- [Telegram](/en/channels/telegram) (el más rápido — solo un token de bot)
- [Discord](/en/channels/discord)
- [Todos los canales](/en/channels)

## Copias de seguridad y migración

Exporte su configuración y espacio de trabajo:

```bash
openclaw backup create
```

Esto crea un archivo de copia de seguridad portable que puede restaurar en cualquier host de OpenClaw. Consulte [Copia de seguridad](/en/cli/backup) para obtener detalles.
