title: Despliega en Railway
---

Despliega OpenClaw en Railway con una plantilla de un solo clic y termina la configuración en tu navegador.
Esta es la ruta más fácil "sin terminal en el servidor": Railway ejecuta el Gateway por ti,
y tú configuras todo a través del asistente web `/setup`.

## Lista de verificación rápida (nuevos usuarios)

1. Haga clic en **Implementar en Railway** (abajo).
2. Añada un **Volumen** montado en `/data`.
3. Establezca las **Variables** requeridas (al menos `SETUP_PASSWORD`).
4. Habilite el **Proxy HTTP** en el puerto `8080`.
5. Abra `https://<your-railway-domain>/setup` y finalice el asistente.

## Implementación con un clic

</a>

Después de la implementación, encuentre su URL pública en **Railway → su servicio → Configuración → Dominios**.

Railway le dará:

- un dominio generado (a menudo `https://<something>.up.railway.app`), o
- usará su dominio personalizado si adjuntó uno.

Luego abra:

- `https://<your-railway-domain>/setup` — configuración web (protegida con contraseña)
- `https://<your-railway-domain>/openclaw` — Interfaz de control

## Lo que obtiene

- Gateway OpenClaw alojado + Interfaz de control
- Configuración web en `/setup` (sin comandos de terminal)
- Almacenamiento persistente a través de Railway Volume (`/data`) para que la configuración/credenciales/espacio de trabajo sobrevivan a las reimplemantaciones
- Exportación de copia de seguridad en `/setup/export` para migrar fuera de Railway más tarde

## Configuración requerida en Railway

### Redes públicas

Habilite el **Proxy HTTP** para el servicio.

- Puerto: `8080`

### Volumen (requerido)

Adjunte un volumen montado en:

- `/data`

### Variables

Establezca estas variables en el servicio:

- `SETUP_PASSWORD` (requerido)
- `PORT=8080` (requerido: debe coincidir con el puerto en Redes públicas)
- `OPENCLAW_STATE_DIR=/data/.openclaw` (recomendado)
- `OPENCLAW_WORKSPACE_DIR=/data/workspace` (recomendado)
- `OPENCLAW_GATEWAY_TOKEN` (recomendado; trátelo como un secreto de administrador)

## Flujo de configuración

1. Visite `https://<your-railway-domain>/setup` e introduzca su `SETUP_PASSWORD`.
2. Elija un proveedor de modelo/autenticación y pegue su clave.
3. (Opcional) Añada tokens de Telegram/Discord/Slack.
4. Haga clic en **Ejecutar configuración**.

Si los mensajes directos de Telegram están configurados para emparejamiento, la configuración web puede aprobar el código de emparejamiento.

## Obtener tokens de chat

### Token del bot de Telegram

1. Mensaje `@BotFather` en Telegram
2. Ejecutar `/newbot`
3. Copia el token (parece `123456789:AA...`)
4. Pégalo en `/setup`

### Token del bot de Discord

1. Ir a [https://discord.com/developers/applications](https://discord.com/developers/applications)
2. **New Application** → elegir un nombre
3. **Bot** → **Add Bot**
4. **Enable MESSAGE CONTENT INTENT** en Bot → Privileged Gateway Intents (requerido o el bot fallará al iniciarse)
5. Copia el **Bot Token** y pégalo en `/setup`
6. Invita al bot a tu servidor (OAuth2 URL Generator; ámbitos: `bot`, `applications.commands`)

## Copias de seguridad y migración

Descargar una copia de seguridad en:

- `https://<your-railway-domain>/setup/export`

Esto exporta tu estado y espacio de trabajo de OpenClaw para que puedas migrar a otro host sin perder la configuración ni la memoria.
