Implemente OpenClaw en Northflank con una plantilla de un solo clic y termine la configuración en su navegador.
Esta es la ruta más fácil "sin terminal en el servidor": Northflank ejecuta la Gateway por usted,
y usted configura todo a través del asistente web `/setup`.

## Cómo empezar

1. Haga clic en [Deploy OpenClaw](https://northflank.com/stacks/deploy-openclaw) para abrir la plantilla.
2. Cree una [cuenta en Northflank](https://app.northflank.com/signup) si aún no tiene una.
3. Haga clic en **Deploy OpenClaw now**.
4. Establezca la variable de entorno requerida: `SETUP_PASSWORD`.
5. Haga clic en **Deploy stack** para compilar y ejecutar la plantilla de OpenClaw.
6. Espere a que se complete la implementación y luego haga clic en **View resources**.
7. Abra el servicio OpenClaw.
8. Abra la URL pública de OpenClaw y complete la configuración en `/setup`.
9. Abra la interfaz de usuario de control en `/openclaw`.

## Lo que obtiene

- OpenClaw Gateway alojado + Interfaz de usuario de control
- Configuración web en `/setup` (sin comandos de terminal)
- Almacenamiento persistente a través de Northflank Volume (`/data`) para que la configuración/credenciales/espacio de trabajo sobrevivan a las reimplementaciones

## Flujo de configuración

1. Visite `https://<your-northflank-domain>/setup` e ingrese su `SETUP_PASSWORD`.
2. Elija un proveedor de modelo/autenticación y pegue su clave.
3. (Opcional) Agregar tokens de Telegram/Discord/Slack.
4. Haga clic en **Run setup**.
5. Abra la interfaz de usuario de control en `https://<your-northflank-domain>/openclaw`

Si los MD de Telegram están configurados para emparejamiento, la configuración web puede aprobar el código de emparejamiento.

## Obtener tokens de chat

### Token del bot de Telegram

1. Envíe un mensaje a `@BotFather` en Telegram
2. Ejecute `/newbot`
3. Copie el token (se parece a `123456789:AA...`)
4. Péguelo en `/setup`

### Token del bot de Discord

1. Vaya a [https://discord.com/developers/applications](https://discord.com/developers/applications)
2. **New Application** → elija un nombre
3. **Bot** → **Add Bot**
4. **Active MESSAGE CONTENT INTENT** en Bot → Privileged Gateway Intents (requerido o el bot fallará al iniciarse)
5. Copie el **Bot Token** y péguelo en `/setup`
6. Invite el bot a su servidor (Generador de URL de OAuth2; ámbitos: `bot`, `applications.commands`)
