Déployez OpenClaw sur Railway avec un modèle en un clic et terminez la configuration dans votre navigateur.
C'est le chemin le plus simple « sans terminal sur le serveur » : Railway exécute le Gateway pour vous,
et vous configurez tout via l'`/setup` assistant Web.

## Liste de contrôle rapide (nouveaux utilisateurs)

1. Cliquez sur **Déployer sur Railway** (ci-dessous).
2. Ajoutez un **Volume** monté sur `/data`.
3. Définissez les **Variables** requises (au moins `SETUP_PASSWORD`).
4. Activez le **Proxy HTTP** sur le port `8080`.
5. Ouvrez `https://<your-railway-domain>/setup` et terminez l'assistant.

## Déploiement en un clic

<a href="https://railway.com/deploy/clawdbot-railway-template" target="_blank" rel="noreferrer">
  Déployer sur Railway
</a>

Après le déploiement, trouvez votre URL publique dans **Railway → votre service → Paramètres → Domaines**.

Railway effectuera soit :

- vous fournir un domaine généré (souvent `https://<something>.up.railway.app`), ou
- utiliser votre domaine personnalisé si vous en avez attaché un.

Ensuite, ouvrez :

- `https://<your-railway-domain>/setup` — configuration Web (protégée par mot de passe)
- `https://<your-railway-domain>/openclaw` — Interface de contrôle

## Ce que vous obtenez

- OpenClaw Gateway hébergé + Interface de contrôle
- Configuration Web sur `/setup` (pas de commandes terminal)
- Stockage persistant via le Volume Railway (`/data`) afin que la configuration/les informations d'identification/l'espace de travail survivent aux redéploiements
- Exportation de sauvegarde sur `/setup/export` pour migrer hors de Railway plus tard

## Paramètres Railway requis

### Réseau public

Activez le **Proxy HTTP** pour le service.

- Port : `8080`

### Volume (requis)

Attachez un volume monté sur :

- `/data`

### Variables

Définissez ces variables sur le service :

- `SETUP_PASSWORD` (requis)
- `PORT=8080` (requis — doit correspondre au port dans Réseau public)
- `OPENCLAW_STATE_DIR=/data/.openclaw` (recommandé)
- `OPENCLAW_WORKSPACE_DIR=/data/workspace` (recommandé)
- `OPENCLAW_GATEWAY_TOKEN` (recommandé ; à traiter comme un secret administrateur)

## Flux de configuration

1. Visitez `https://<your-railway-domain>/setup` et entrez votre `SETUP_PASSWORD`.
2. Choisissez un modèle/fournisseur d'authentification et collez votre clé.
3. (Facultatif) Ajoutez les jetons Telegram/Discord/Slack.
4. Cliquez sur **Run setup**.

Si les DMs Telegram sont réglés sur l'appairage, la configuration Web peut approuver le code d'appairage.

## Connect a channel

Collez votre jeton Telegram ou Discord dans l'assistant `/setup`. Pour les instructions de configuration, consultez la documentation du channel :

- [Telegram](/fr/channels/telegram) (le plus rapide —仅需 un jeton de bot)
- [Discord](/fr/channels/discord)
- [All channels](/fr/channels)

## Backups & migration

Téléchargez une sauvegarde à l'adresse :

- `https://<your-railway-domain>/setup/export`

Cela exporte votre état OpenClaw + votre espace de travail afin que vous puissiez migrer vers un autre hôte sans perdre la configuration ni la mémoire.
